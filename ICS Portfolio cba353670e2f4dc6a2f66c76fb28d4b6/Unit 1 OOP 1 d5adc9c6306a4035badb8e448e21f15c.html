The opposite of OOP is known as functional programming, where everything is done in one class. No objects are required.

Objects

OOP is meant to recreate real life objects. Objects have methods and attributes. Some methods of a football object could be throwing the football, catching the football, and fixing the football. Some attributes of the football methods could be color, shape, weight, and price. When we change the attributes, we perform actions on the object. Obviously, the object has to exist in the first place, so we instantiate an object.

A class in Java is a blueprint for an object. We use classes to create objects in Java. For example, if we had a Football class, we could do:

Football myBall = new FootBall();

We are making a new Football object called myBall.

Static attributes and methods are not shared by a class‚Äôs objects. They are NOT a part of OOP. Thus, a static method can call only other static methods and can only access static variables.

Encapsulation

When we‚Äôre coding, we want our data and information to be as safe as possible. That‚Äôs why we have encapsulation! If we wanted information in the form of attributes to only be accessible in one class and in no other classes, we would declare that attribute(s) as private. On the other hand, if we didn‚Äôt care if other classes could access this information, we would make it public, thus making the attributes editable and viewable. Nrmally, all instance variables (attributes) are private for security reasons, unless there exists a very good reason for it to be public.

Now, here‚Äôs a complete class with all information that we discussed, summarized.

The class that I will show here is a class that I commented myself as a project. It has methods and attributes. We use OOP here to show that the class can be made as an object which represents a car in real life.

public class Car {
   private String brand;
   private String model;
   private int kilometres;
   private double gas;
   private double tankCapacity;
   // This method is the constructor, which is called whenever an instance of the class is made.
   // A car is defined by its brand, its model, its gas measure, and its tank capacity when first initialized.
   public Car(String b, String m, double g, double tCap){
      brand = b;
      model = m;
      kilometres = 0;
      gas = g;
      tankCapacity = tCap;
   }
   // This method models the car to drive a distance given as a double.
   // When the car drives several of its attributes change, like the total kilometers as well as the gas level which decreases by the distance divided by 5.
   public void drive(double distance){
      kilometres += distance;
      gas -= distance / 5;
   }
   // This method models a car getting refueled by increasing its gas parameter to the max (the attribute tankCapacity is the max).
   public void gasUp(){
      gas = tankCapacity;
   }
} // Car class

This example shows the various methods, like gasUp as well as how they interact with the attributes, making it a prime example of Object Oriented Programming.
