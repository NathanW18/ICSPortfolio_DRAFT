Unit 4: ArrayList

NAVIGATION

Let‚Äôs say that we wanted to store the names and final marks of twenty students in a class. We could use an array. But what if, halfway through the semester, some students came into the class, some students left the class, some marks changed, the principal made a new rule where the marks had to be shuffled, etc.? All of that would be pretty challenging to implement with an array.

Instead, there is another data structure made for the purpose of easy manipulation: ArrayList. ArrayLists are dynamic, where you can add, swap, delete, and insert elements in the same ArrayList. Even better, ArrayLists don‚Äôt all have to include the same data type, unlike arrays. They can even include nested ArrayLists!

Differences Between Arrays and ArrayLists

ArraysArrayLists
Length cannot be changedLength can be changed
Can only hold one data type at a timeCan hold elements of several data types (String, int, etc.)
Uses [] to access elementsUses .get(index) to access elements
Can only hold primitive and object elementsCan only hold object elements. Autoboxes primitive elements to convert them to object elements (for example, from int to Integer).

ArrayList Methods

As mentioned previously, ArrayLists have so many more capabilities than arrays do, including inserting elements between other elements. Here is a short list of some methods that we can perform on ArrayLists.

MethodParameterReturnPurpose

Array Demonstrations

I could use my understanding of ArrayLists, how they work, and ArrayList methods in this assignment called a round of bridge.

Basically, the program I wrote simulates a round of bridge (the game), randomly deals cards out, and adds up the points of the player and finally display the winner. We decided to use ArrayLists because we had to 1) shuffle the elements and 2) add and remove elements easily. Arrays are not capable of performing these functions easily at the same time so we decided to use ArrayLists.

	// Given the String in storage this method adds it to the arraylist and shuffle it to create random cards
   public void shuffle ()
   {
      deck = new ArrayList < String > (); // initialize the ArrayList to contain Strings
      for (int i = 0 ; i < totalCards.length () ; i += 3)
      {
         String str = totalCards.substring (i, i + 2);
         deck.add (str); // loops through deck and adds to ArrayList
      }
      Collections.shuffle (deck); // randomly shufles ArrayList
   }

What we did here is we shuffled the cards. Based on the cards String which the user inputted, we then added those Strings to the ArrayList using the add() method. Then, we shuffled the ArrayList using Collections.shuffle().

Here, we decided to store Strings inside the ArrayLists, but they could store anything else that is an Object, and even different data types! Needless to say, ArrayLists are so much more cooler than Arrays!

Common Mistakes

  1. Mixing Array and ArrayList syntax up
    1. Result: Compilation errors. Check the ArrayList methods table to refer to which methods are exclusively for ArrayLists.
  1. Initializing an ArrayList with a primitive data type
    1. Result: Compilation errors. Refer to the autoboxing section above.
  1. Looping through an ArrayList with for loop and adding/removing elements
    1. Result: Not all elements are accessed correctly. Use a while loop instead. Refer to the above tutorial and second code snippet.
